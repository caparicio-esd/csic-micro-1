## Objetivo

Proyecto end-to-end pequeño pero completo sobre un problema de la propia disciplina del participante, que recorre todo el módulo.

## Fases

1. **Definir** pregunta, dato necesario y plan _(bloque 5)_
2. **Obtener y preparar** el dataset, documentando calidad y descartando fugas _(bloque 2)_
3. **Modelar y evaluar** con métricas honestas y diagnóstico de overfitting _(bloque 3)_, o usar/evaluar un modelo preentrenado/LLM _(bloque 4)_
4. **Empaquetar** de forma reproducible y, opcionalmente, exponer como API mínima _(bloque 6)_

## Entregable

- Repositorio reproducible (ejecuta de arriba a abajo)
- Memoria breve (2–3 págs.): problema, decisiones de datos, modelo, resultados, limitaciones

## Vías

| Vía | Descripción | Énfasis |
|-----|-------------|---------|
| **Modelo propio** | Entrenas algo simple end-to-end | Bloques 2–3 |
| **Modelo preentrenado** | Usas y evalúas un modelo/LLM existente | Bloques 2, 4, 6 |

## Criterios de evaluación

- Reproducibilidad (ejecuta en otra máquina)
- Honestidad metodológica (métricas adecuadas, fuga descartada, limitaciones declaradas)
- Justificación de decisiones
- Estructura de proyecto limpia
- El problema es real para el participante
